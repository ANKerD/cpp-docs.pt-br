# <a name="windows-platforms-crt"></a>Plataformas Windows (CRT)

As bibliotecas de tempo de execução do C para Visual Studio são compatíveis com as versões atuais do Windows e do Windows Server, [!INCLUDE[win8](../build/reference/includes/win8_md.md)], [!INCLUDE[winserver8](../build/reference/includes/winserver8_md.md)], [!INCLUDE[win7](../build/includes/win7_md.md)], [!INCLUDE[winsvr08](../build/reference/includes/winsvr08_md.md)], com o Windows Vista e, opcionalmente, com o [!INCLUDE[winxp](../build/includes/winxp_md.md)] SP3 (Service Pack 3) para x86, o [!INCLUDE[winxp](../build/includes/winxp_md.md)] SP2 (Service Pack 2) para x64 e o [!INCLUDE[WinXPSvr](../build/includes/winxpsvr_md.md)] SP2 (Service Pack 2) para x86 e x64. Todos esses sistemas operacionais são compatíveis com a API de área de trabalho do Windows (Win32) e com Unicode. Além disso, todos os aplicativos Win32 podem usar um conjunto de caracteres multibyte (MBCS).

> [!NOTE]
> A instalação padrão da carga de trabalho **Desenvolvimento para desktop com C++** no Visual Studio 2017 não é compatível com o desenvolvimento para [!INCLUDE[winxp](../build/includes/winxp_md.md)] e [!INCLUDE[WinXPSvr](../build/includes/winxpsvr_md.md)]. Você precisa instalar o componente opcional **Suporte do Windows XP para C++** para habilitar um conjunto de ferramentas da plataforma Windows XP.
