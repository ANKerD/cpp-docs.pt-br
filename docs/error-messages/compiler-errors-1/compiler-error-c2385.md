# <a name="compiler-error-c2385"></a>C2385 de erro do compilador
acesso ambíguo de 'member'  
  
 O membro pode derivar de mais de um objeto (herdado de mais de um objeto).  Para resolver esse erro,  
  
-   Torne o membro inequívoca fornecendo uma conversão.  
  
-   Renomear os membros ambíguos nas classes base.  
  
## <a name="example"></a>Exemplo  
 O exemplo a seguir gera C2385.  
  
```  
// C2385.cpp  
// C2385 expected  
#include <stdio.h>  
  
struct A   
{  
    void x(int i)   
    {  
        printf_s("\nIn A::x");  
    }  
};  
  
struct B   
{  
    void x(char c)   
    {  
        printf_s("\nIn B::x");  
    }  
};  
  
// Delete the following line to resolve.  
struct C : A, B {}  
  
// Uncomment the following 4 lines to resolve.  
// struct C : A, B   
// {  
//     using B::x;  
//     using A::x;  
// };  
  
int main()   
{  
    C aC;  
    aC.x(100);  
    aC.x('c');  
}  
  
struct C : A, B   
{  
    using B::x;  
    using A::x;  
};  
```
