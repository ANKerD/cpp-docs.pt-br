# Aviso do compilador (n&#237;vel 1) C4346
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

“name”: o nome dependente não é um tipo  
  
 A palavra\-chave de [typename](../Topic/typename.md) é necessário se um nome dependente deve ser tratado como um tipo.  Essa é uma alteração no compilador do Visual C\+\+ .NET 2003, feito para estar em conformidade com o padrão ISO C\+\+.  
  
 Para o código que funciona da mesma maneira em todas as versões do Visual C\+\+, adicione `typename` à declaração.  
  
 O seguinte exemplo gera C4346:  
  
```  
// C4346.cpp  
// compile with: /WX /LD  
template<class T>  
struct C {  
   T::X* x;   // C4346  
   // try the following line instead  
   // typename T::X* x;  
};  
```  
  
 Os exemplos a seguir mostram a outros exemplos em que a palavra\-chave de **typename** é necessário:  
  
```  
// C4346b.cpp  
// compile with: /LD /W1  
template<class T>  
const typename T::X& f(typename T::Z* p);   // Required in both places  
  
template<class T, int N>  
struct L{};  
  
template<class T>  
struct M : public L<typename T::Type, T::Value>   
{   // required on type argument, not on non-type argument  
   typedef typename T::X   Type;  
   Type f();   // OK: "Type" is a type-specifer  
   typename T::X g();   // typename required  
   operator typename T::Z();   // typename required      
};  
```  
  
 e isso,  
  
```  
// C4346c.cpp  
// compile with: /LD /WX  
struct Y {  
   typedef int Y_t;  
};  
  
template<class T>  
struct A {  
   typedef Y A_t;  
};  
  
template<class T>  
struct B {  
   typedef /*typename*/ A<T>::A_t B_t;   // C4346 typename needed here  
   typedef /*typename*/ B_t::Y_t  B_t2;   // typename also needed here  
};  
```
