# <a name="writing-mfc-applications"></a>Escrevendo aplicativos MFC
Este artigo explica as etapas iniciais para desenvolver seu aplicativo. Primeiro, você deve decidir que tipo de aplicativo que você está escrevendo. Várias das opções discutidas [escolhas de Design de aplicativo](../mfc/application-design-choices.md). Seu aplicativo será:  
  
-   Em execução na Internet ou intranet  
  
-   Em execução em um cliente ou em um servidor  
  
-   Em execução em um navegador ou como um aplicativo autônomo  
  
-   Usando a tecnologia COM ou ativo  
  
-   Baixando dados usando WinInet ou monikers assíncronos  
  
 Suas decisões de determinam quais classes são apropriadas para seu aplicativo. As respostas também ajudam a determinar as seleções feitas quando você executa o Assistente de aplicativo para começar a criar seu aplicativo.  
  
 Depois de fazer suas decisões de design inicial sobre o seu aplicativo de Internet, você pode usar o Assistente de aplicativo para começar. Use o Assistente de aplicativo para criar um aplicativo de esqueleto e modificar o código, conforme descrito nos seguintes artigos:  
  
-   Para um controle ActiveX, consulte [controles ActiveX na Internet](../mfc/activex-controls-on-the-internet.md).  
  
-   Para um documento ativo, consulte [documentos ativos na Internet](../mfc/active-documents-on-the-internet.md).  
  
 Os artigos a seguir também fornecem instruções para ajudar você a começar suas tarefas de programação:  
  
-   [Opções de design do aplicativo](../mfc/application-design-choices.md)  
  
-   [Monikers assíncronos na Internet](../mfc/asynchronous-monikers-on-the-internet.md)  
  
-   [Noções básicas de WinInet](../mfc/wininet-basics.md)
