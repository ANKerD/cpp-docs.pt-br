# Suporte &#224; biblioteca de v&#237;nculo din&#226;mico
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

As bibliotecas de NAFXCW.lib e de NAFXCWD.lib \(listadas na tabela convenções de nomenclatura da biblioteca de Estática\- link em [Convenções de nomenclatura de biblioteca](../Topic/Library%20Naming%20Conventions.md)\) cria o projeto como uma biblioteca de vínculo dinâmico\) nativa, chamada “DLL normal \(anteriormente” um “USRDLL”\) que pode ser usado com os aplicativos não compilados com a biblioteca da classe.  Esse suporte a DLL não é o mesmo que MFCx0.DLL e MFCx0D.DLL \(conhecidos como AFXDLL\), que contêm toda a biblioteca de classes em uma DLL.  Para obter mais informações, consulte [Dlls](../build/dlls-in-visual-cpp.md).  Para uma tabela de nomes de DLL, consulte [Convenções de nomenclatura para dlls MFC](../build/naming-conventions-for-mfc-dlls.md).
