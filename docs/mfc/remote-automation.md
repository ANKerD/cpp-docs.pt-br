# <a name="remote-automation"></a>Automação remota
> [!NOTE]
>  Automação remota é uma tecnologia obsoleta e não deve ser usada para novos aplicativos. Visual C++ não dá suporte para o Windows 95. Casos em que você precisa dar suporte a automação remota são descritos na [onde Does remoto automação se ajustar no](where-does-remote-automation-fit-in-q.md).  
  
 Automação remota é um tipo de [automação](../mfc/automation.md) que permite que um consumidor de interface executar um provedor de interface que reside em outro computador, por exemplo, em uma rede.  
  
 Este artigo explica como criar objetos de automação que podem ser chamados e executados remotamente e como criar controladores de automação que podem usar esses objetos de automação remota. Ele também examina as opções de configuração e destaca as principais diferenças entre a automação remota e DCOM (a versão distributed COM e OLE que permite que as interfaces, exceto aquelas relacionadas à automação a ser chamado e executado remotamente).
